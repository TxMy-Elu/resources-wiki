resources_front · déployé sur Vercel
Stack technique
Next.js 16 React 18 TypeScript Tailwind CSS pnpm 9
Rendu
SSR (Server-Side Rendering) — les pages sont pré-rendues côté serveur Vercel pour le SEO et la performance. Certaines routes sont client-only (dashboard).

Vercel — déploiement automatique sur chaque push main. Chaque PR génère une URL de prévisualisation unique.

Appels HTTPS vers l'API Symfony avec Authorization: Bearer <JWT>. Variables d'env : NEXT_PUBLIC_API_URL, NEXT_PUBLIC_SUPABASE_URL.

Nginx — Reverse Proxy & Terminaison SSL
resources_nginx · nginx:alpine · seul point d'entrée exposé vers Internet
Rôle principal
Reverse proxy : reçoit toutes les requêtes entrantes et les route. Gère le chiffrement TLS — PHP reçoit les requêtes en HTTP simple en interne, ce qui simplifie la config Symfony.
Certificat SSL
Let's Encrypt — renouvellement automatique via Certbot (cron) tous les 90 jours. TLS 1.2 minimum, TLS 1.0/1.1 désactivés.
Ports exposés
:443 → HTTPS prod :8080 → HTTP dev
Tous les autres ports (MySQL :3306, PHP-FPM :9000) sont invisibles depuis Internet.
Protocole vers PHP
FastCGI sur resources_php:9000 (réseau bridge Docker interne). Nginx passe les requêtes à PHP-FPM, récupère la réponse, la retransmet au client.
Fichiers statiques
Nginx sert les assets statiques (images, CSS, JS) directement sans passer par PHP — 10× plus rapide. PHP n'est appelé que pour la logique métier.
Image
nginx:alpine — ~25 MB. Alpine Linux pour la surface d'attaque minimale.
⚙️
API REST — Symfony 8 · PHP 8.4
ressources_api · php:8.4-fpm-alpine · 44 endpoints · RBAC 4 rôles
Stack technique
Symfony 8 PHP 8.4-fpm Doctrine ORM LexikJWTBundle NelmioCorsBundle SonarCloud
Authentification
JWT RS256 — paire RSA-2048. TTL 3600s. Refresh token en base. Clés PEM dans GitHub Secrets, jamais commitées.
Contrôle d'accès (RBAC)
ROLE_CONNECTED ROLE_MODERATOR ROLE_ADMIN ROLE_SUPER_ADMIN
Hiérarchie Symfony + Voters pour le contrôle au niveau de l'entité (auteur d'une ressource). Catch-all : tout /api/** non listé requiert IS_AUTHENTICATED_FULLY.
Sécurité conteneur
Utilisateur non-root symfony (UID 1001). Image multi-stage : stage builder (composer) → stage runtime (php-fpm-alpine ~50 MB). HEALTHCHECK toutes les 30s.
Endpoints principaux
44 endpoints REST : auth (login, refresh, reset), ressources (CRUD), catégories, utilisateurs (admin), modération, statistiques, logs.
Flux JWT
POST /api/login→ bcrypt verify→ JWT signé RS256→ Bearer header

MySQL 8.0 — Base de données relationnelle
resources_db · Docker · volume persistant db_data · port 3306 (interne uniquement)
Isolation réseau
Port 3306 accessible uniquement depuis le réseau bridge Docker interne. En production : aucun port MySQL exposé vers Internet. En dev : 3307→3306 pour DBeaver/TablePlus.
Persistance
Volume Docker nommé db_data monté sur /var/lib/mysql. Si le conteneur est supprimé et recréé, les données survivent dans le volume.

Sauvegardes
Dump mysqldump quotidien. RPO = 24h. Rétention 30 jours. Stocké hors du serveur de production.
Monitoring
MySQL Exporter → Prometheus. Métriques : QPS, connexions actives, InnoDB buffer pool hit rate, slow queries.

Supabase Storage — Stockage des médias
Fichiers images et documents · bucket Ressources · CDN public · S3-compatible
Pourquoi pas un volume Docker ?
Un volume local crée des problèmes de scalabilité (non partagé si plusieurs serveurs), de sauvegarde et de CDN. Supabase gère la réplication, la CDN mondiale et les sauvegardes automatiquement.
Flux d'upload
Fichier (multipart)→Validation Symfony (MIME, taille)
SDK Supabase PHP → bucket→URL CDN publique→Stocké dans url_media
Sécurité
Clé service_role (accès écriture) stockée dans GitHub Secrets. Clé anon (lecture publique) acceptable côté client. Jamais de clé dans le code.
Types de fichiers
Images de couverture Avatars utilisateurs Documents PDF

Stack Monitoring — Prometheus · Grafana · Loki
docker-compose.monitoring.yaml · données conservées 30 jours
Prometheus :9090
Collecte les métriques en mode pull (scrape toutes les 15s). Sources : API Symfony (/metrics), Node Exporter (système), MySQL Exporter. Évalue les règles d'alerting.
Grafana :3001
Dashboard visuel en temps réel : temps de réponse p95, taux d'erreurs, CPU/RAM, requêtes BDD. Alertes automatiques par email si seuil dépassé. Agrège métriques (Prometheus) + logs (Loki).
Loki + Promtail
Loki stocke et indexe les logs (alternative légère à Elasticsearch). Promtail lit les logs Docker et les envoie à Loki. Corrélation log/métrique dans Grafana pour le débogage.
